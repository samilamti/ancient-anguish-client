# v5.0 Release Notes

**Date:** March 16, 2026
**Previous release:** v4.1 (March 15, 2026)

---

## Highlights

Web client support! Ancient Anguish can now be played in a browser. A companion Dart server handles authentication, per-user file storage, MUD connection proxying, and audio streaming. All existing features (triggers, aliases, area config, themes, command history) work on both desktop and web.

---

## New Features

### Web Client
- Play Ancient Anguish in any modern browser — no install required
- JWT-based authentication with login/register flow
- Per-user profiles: settings, triggers, aliases, and area config stored server-side
- WebSocket-to-TCP proxy relays MUD traffic through the companion server
- HTML5 Audio API for area music playback with crossfade support
- All four themes (RPG Fantasy, Classic Dark, High Contrast, Custom) available on web

### Web Companion Server
- Dart Shelf server (`server/`) providing backend services
- JWT authentication with bcrypt password hashing
- Per-user file storage API with whitelist-controlled access
- WebSocket MUD proxy (raw byte relay, client-side telnet parsing)
- Audio file streaming with auth tokens
- CORS middleware for local development
- 57 server tests covering auth, files, proxy, JWT, and whitelist

### Service Abstraction Layer
- All platform-dependent services behind interfaces: `StorageService`, `MudConnectionService`, `AudioInterface`
- Compile-time platform selection via Dart conditional imports
- Desktop implementations unchanged; web implementations added alongside
- Platform stubs for dart:io-dependent code (file utilities, exit handling, CWD migration)

### Settings Persistence
- Settings now saved to `settings.json` on every change
- Loaded at app initialization via StorageService
- Works on both desktop (local file) and web (server API)
- Theme, font, scrollback, social windows, emoji parsing — all persisted

---

## Platform Guards

- Social windows: desktop-only (hidden on web)
- Area config file picker: desktop-only (web uses server-side files)
- App initialization order: auth required before storage access on web

---

## Technical Details

- New dependencies: `http`, `web`, `web_socket_channel`
- Flutter web platform enabled (`flutter create --platforms web`)
- Compile-time `SERVER_URL` override via `--dart-define`
- 667 client tests passing, 57 server tests passing, 0 analysis issues
