# v6.1 Release Notes

**Date:** March 20, 2026
**Previous release:** v6.0 (March 20, 2026)

---

## Highlights

Party messages in social windows, configurable input line wrap for accessibility, and alias auto-loading at login.

---

## New Features

### Party Messages Social Tab
- New third tab in the social windows system captures party channel messages
- Regex parser matches `<Party Name> [Character] : message` format
- Display strips the party name prefix, showing only `Character: message`
- Send party messages via `pt <msg>` from the input bar
- Per-day history files in `Party History/` folder
- Unread indicators (amber dot) when the party tab is inactive

### Accessibility: Input Line Wrap
- New "Accessibility" section in Settings with an input line wrap slider (0–200 characters)
- When enabled, constrains the input field width so text wraps at the configured column
- Visual only — whole words are kept together and the sent text is unaffected
- Set to 0 (Off) by default

---

## Improvements

### Alias Loading at Login
- Aliases now load automatically after character login
- Previously required visiting the Alias settings screen to trigger lazy provider initialization

### Audio Controls Cleanup
- Removed the area name label from the audio controls bar for a cleaner look

### Social Windows Default Floating
- Social panels now start in floating mode instead of auto-docking to the right edge
- Removed edge-snap auto-dock behavior; use the dock button in the title bar to dock manually

---

## Technical Details

- `social_message_parser.dart`: `PartyMatchResult` class + `matchPartyLine()` with regex `^<([^>]+)>\s+\[(\w+)\]\s+:\s+(.+)$`
- `social_panel_state.dart`: `partyPanel` + `partyHasUnread` fields in `SocialWindowsState`
- `connection_provider.dart`: `_markUnreadIfInactive()` refactored to `{required int tabIndex}` for 3-tab support
- `input_bar.dart`: `_measureCharWidth()` + `ConstrainedBox` for column-based wrap
- `login_provider.dart`: `ref.read(aliasRulesProvider)` in `_performPostLogin()` forces lazy init
- 676 client tests passing, 62 server tests passing, 0 analysis errors
